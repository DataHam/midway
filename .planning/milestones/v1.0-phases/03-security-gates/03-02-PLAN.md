<task type="auto">
  <name>Task 2: Implement Danny API endpoint</name>
  <files>src/api/verify-danny/index.js</files>
  <action>
Implement Danny verification endpoint with full flow:

1. **Request parsing**:
   - Read request body as JSON
   - Extract token and sitekey from body
   - Parse redirect URL from query parameter (default: https://danny.tamtham.com)
   - Get client IP from request headers
   - Get user agent from request headers

2. **Rate limit check** (use shared utility):
   - Call checkRateLimit(request)
   - If not allowed:
     - Log rate_limit event
     - Return 429 with RATE_LIMITED error

3. **Sitekey validation**:
   - Call validateSitekey(sitekey)
   - If invalid: log warning, continue (sitekey is public, minor misconfiguration)

4. **Cloudflare Siteverify API call**:
   - POST to https://api.cloudflare.com/client/v4/turnstile/verify
   - Body: { secret: getSecret(), response: token, remoteip: clientIP }
   - Handle network errors → SERVICE_UNAVAILABLE
   - Parse response: success (boolean), error-codes (array)

5. **Response handling**:
   - If API success:
     - Log success event with userId: "danny"
     - Return 302 redirect to redirect URL
   - If API failure:
     - Record failed attempt (increment cookie)
     - Map error-codes to error codes:
       - "missing-secret" → SERVICE_UNAVAILABLE
       - "invalid-secret" → SERVICE_UNAVAILABLE
       - "expired" → EXPIRED_TOKEN
       - "invalid" → INVALID_CODE
       - "rate-limited" → RATE_LIMITED
     - Log failure event with errorCode
     - Return 403 with JSON error response

6. **Response format**:
   - Browser (user-agent contains Mozilla): 302 redirect with Location header
   - API (Accept: application/json): JSON response
   - JSON success: { success: true, redirectUrl: string }
   - JSON failure: { success: false, error: errorCode, message: humanMessage }

Reference Cloudflare Siteverify API docs. Use fetch for API call. Handle all error cases. Log all events.
</action>
