<task type="auto">
  <name>Task 2: Implement Helen API endpoint</name>
  <files>src/api/verify-helen/index.js</files>
  <action>
Implement Helen verification endpoint (identical pattern to Danny API):

1. **Request parsing**:
   - Read request body as JSON
   - Extract token and sitekey from body
   - Parse redirect URL from query parameter (default: https://helen.tamtham.com)
   - Get client IP from request headers
   - Get user agent from request headers

2. **Rate limit check**:
   - Call checkRateLimit(request)
   - If not allowed: log rate_limit event, return 429 with RATE_LIMITED error

3. **Sitekey validation**:
   - Call validateSitekey(sitekey)
   - Log warning if invalid (continue execution)

4. **Cloudflare Siteverify API call**:
   - POST to https://api.cloudflare.com/client/v4/turnstile/verify
   - Body: { secret: getSecret(), response: token, remoteip: clientIP }
   - Handle network errors → SERVICE_UNAVAILABLE
   - Parse response: success (boolean), error-codes (array)

5. **Response handling**:
   - If API success:
     - Log success event with userId: "helen"
     - Return 302 redirect to redirect URL
   - If API failure:
     - Record failed attempt (increment cookie)
     - Map error-codes to error codes (same as Danny)
     - Log failure event with errorCode
     - Return 403 with JSON error response

6. **Response format**:
   - Browser: 302 redirect with Location header
   - API: JSON response
   - JSON success: { success: true, redirectUrl: string }
   - JSON failure: { success: false, error: errorCode, message: humanMessage }

Key differences from Danny API:
- Redirect URL defaults to https://helen.tamtham.com
- userId in logs is "helen"

Reference 03-02 SUMMARY for implementation pattern. Copy structure, change redirect URL and userId.
</action>
