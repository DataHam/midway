<task type="auto">
  <name>Task 1: Create GitHub Actions workflow for Azure SWA deployment</name>
  <files>.github/workflows/azure-static-web-apps.yml</files>
  <action>
    Create `.github/workflows/azure-static-web-apps.yml` with the following structure:

    **Workflow configuration:**
    - Name: "Azure Static Web Apps CI/CD"
    - Trigger: `push` to `main` branch, `pull_request` for preview deployments
    - Jobs: `build_and_deploy_job` and `close_pull_request_job`

    **Build job steps:**
    1. Checkout code with `actions/checkout@v4`
    2. Install OIDC client dependencies (`@actions/core@1.10.0`, `@actions/http-client@1.8.11`)
    3. Get OIDC token via `actions/github-script@v7`
    4. Doppler setup using `dopplerhq/action@v2` with project="tamtham-website", config="production"
    5. Build and deploy using `Azure/static-web-apps-deploy@v1` with:
       - `azure_static_web_apps_api_token`: `${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN }}`
       - `action`: "upload"
       - `app_location`: "/"
       - `api_location`: "api"
       - `output_location`: "."
       - `app_build_command`: "npm install && npm run build"
       - `api_build_command`: "cd api && npm install && npm run build"
       - `github_id_token`: OIDC token from step 3

    **Close PR job:**
    - Trigger on PR closed
    - Use `Azure/static-web-apps-deploy@v1` with `action: "close"`

    **Version pinning:**
    - Pin all actions to major version only (e.g., `@v4`, `@v1`, `@v7`) for security updates
    - Do NOT use `@latest` or exact patch versions

    **Build timeout:**
    - Add `build_timeout_in_minutes: 30` to handle Tailwind compilation and image optimization
  </action>
  <verify>
    <automated>cat .github/workflows/azure-static-web-apps.yml | grep -E "@v[0-9]|dopplerhq/action|Azure/static-web-apps-deploy" | wc -l</automated>
    Expected: At least 5 action references with version pinning
  </verify>
  <done>
    Workflow file created with:
    - Push to main triggers deployment
    - Doppler integration for secrets injection
    - All actions pinned to major versions
    - API functions configured in `api/` directory
    - Build command includes Tailwind compilation
  </done>
</task>

    **Section 2: Add Secrets**
    List of required secrets to add:
    - `TURNSTILE_SECRET_KEY`: Cloudflare Turnstile secret key (from Cloudflare Dashboard)
    - `AZURE_SUBSCRIPTION_ID`: Azure subscription ID (from Azure Portal)
    - `AZURE_TENANT_ID`: Azure AD tenant ID (from Azure Portal)
    - `AZURE_CLIENT_ID`: Azure AD application client ID (from Azure Portal)
    - `AZURE_STATIC_WEB_APPS_API_TOKEN`: Azure SWA API token (from Azure Portal -> Static Web Apps -> Deployment credentials)

    **Section 3: Generate Service Token**
    - Navigate to: Workplace -> Settings -> API Tokens
    - Create Service Token with "Read" permissions for CI/CD
    - Copy token and add to GitHub Secrets as `DOPPLER_TOKEN`

    **Section 4: GitHub Actions Secret Setup**
    - Go to: Repository -> Settings -> Secrets and variables -> Actions
    - Add secret: `DOPPLER_TOKEN` (value from Step 3)
    - Add secret: `AZURE_STATIC_WEB_APPS_API_TOKEN` (from Azure Portal)

    **Verification:**
