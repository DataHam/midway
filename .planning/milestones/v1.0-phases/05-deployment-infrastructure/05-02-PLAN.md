    - Sign in to Cloudflare Dashboard
    - Click "Add a Site"
    - Enter domain: `tamtham.com`
    - Select plan: Free (sufficient for this use case)
    - Wait for nameserver scan

    **Section 2: Update Nameservers**
    - Copy Cloudflare nameservers (e.g., `ns1.cloudflare.com`, `ns2.cloudflare.com`)
    - Update at domain registrar (GoDaddy, Namecheap, etc.)
    - Wait for propagation (typically minutes)

    **Section 3: Configure DNS Records**
    Create CNAME record:
    - Type: `CNAME`
    - Name: `tamtham.com` (or `@`)
    - Target: `<your-app-name>.azurestaticapps.net` (replace with actual Azure SWA endpoint)
    - Proxy status: `Proxied` (orange cloud)
    - TTL: Auto

    **Section 4: Enable Bot Fight Mode**
    - Navigate: Cloudflare Dashboard -> tamtham.com -> Security -> WAF
    - Enable "Bot Fight Mode"
    - Note: Automatic detection, no additional configuration needed

    **Section 5: Configure Cache Rules (Modern Approach)**
    Navigate: Cloudflare Dashboard -> tamtham.com -> Caching -> Cache Rules

    Create Rule 1 - Cache static assets:
    - Rule name: "Cache static assets"
    - Field: `URI Path`
    - Operator: `starts with`
    - Value: `/`
    - Action: `Cache`
    - Cache TTL: `1 year`

    Create Rule 2 - Bypass cache for API:
    - Rule name: "Bypass cache for API"
    - Field: `URI Path`
    - Operator: `starts with`
    - Value: `/api`
    - Action: `Bypass Cache`

    **Section 6: Verification**
    - Check DNS: `dig tamtham.com` or `nslookup tamtham.com`
    - Verify proxy: `curl -I https://tamtham.com` should show Cloudflare headers
    - Test cache rules: `curl -I -H "Cache-Control: no-cache" https://tamtham.com`

<task type="auto">
  <name>Task 2: Create Cloudflare Tunnel configuration guide</name>
  <files>.planning/phases/05-deployment-infrastructure/TUNNEL-CONFIG.md</files>
  <action>
    Create `.planning/phases/05-deployment-infrastructure/TUNNEL-CONFIG.md` with Cloudflare Tunnel setup:

    **Section 1: Create Tunnel in Cloudflare Dashboard**
    - Navigate: Cloudflare Dashboard -> Zero Trust -> Tunnels
    - Click "Create a tunnel"
    - Name: `danny-tunnel`
    - Select deployment method: Manual (for home server)
    - Copy credentials JSON or TUNNEL_TOKEN

    **Section 2: Install cloudflared on Home Server**
    For Linux (Ubuntu/Debian):
    ```bash
    curl -L --output cloudflared https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
    chmod +x cloudflared
    sudo mv cloudflared /usr/local/bin/
    ```

    **Section 3: Authenticate Tunnel**
    Option A - Using credentials file:
    ```bash
    cloudflared tunnel login
    # Opens browser for authentication
    # Saves credentials to ~/.cloudflared/credentials.json
    ```

    Option B - Using TUNNEL_TOKEN:
    ```bash
    export TUNNEL_TOKEN="xxx-xxx-xxx"
    cloudflared tunnel create danny-tunnel
    ```

    **Section 4: Configure Tunnel**
    Create `/etc/cloudflared/config.yml`:
    ```yaml
    tunnel: danny-tunnel
    credentials-file: /etc/cloudflared/credentials.json

    ingress:
      - hostname: danny.tamtham.com
        service: http://localhost:8080
        path: /
      - service: http_status:404
    ```

    For both subdomains in one tunnel:
    ```yaml
    tunnel: combined-tunnel
    credentials-file: /etc/cloudflared/credentials.json

    ingress:
      - hostname: danny.tamtham.com
        service: http://localhost:8080
        path: /
      - hostname: helen.tamtham.com
        service: http://localhost:8080
        path: /
      - service: http_status:404
    ```

    **Section 5: Run Tunnel as Systemd Service**
    Create `/etc/systemd/system/cloudflared.service`:
    ```ini
    [Unit]
    Description=Cloudflare Tunnel
    RequiresNetworkOnline.target
    After=network-online.target

    [Service]
    Type=exec
    User=www-data
    ExecStart=/usr/local/bin/cloudflared tunnel --config /etc/cloudflared/config.yml run

    [Install]
    WantedBy=multi-user.target
    ```

    Enable and start:
    ```bash
    sudo systemctl enable cloudflared
    sudo systemctl start cloudflared
    sudo systemctl status cloudflared
    ```

    **Section 6: Verification**
    - Check tunnel status: `cloudflared tunnel list`
    - Check logs: `journalctl -u cloudflared -f`
    - Test access: `curl -I https://danny.tamtham.com`
    - Test access: `curl -I https://helen.tamtham.com`
  </action>
