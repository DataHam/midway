## Phase 05-04: Migrate Verification APIs to Cloudflare Workers

### Objective

Replace Azure Functions with Cloudflare Workers for verification API endpoints, eliminating Azure dependency and Cloudflare tunnel infrastructure.

**Purpose:** Cloudflare Workers run on Cloudflare's edge network, providing faster responses and eliminating the need for both Azure Functions and home server tunnels. Since you're already using Cloudflare for DNS and security, this creates a unified infrastructure.

**Output:** Two Cloudflare Worker scripts (verify-danny, verify-helen) that validate Turnstile tokens and redirect on success.

### Architecture Change

**Before (Azure Functions + Tunnels):**
```
tamtham.com (Azure SWA)
  ↓
/api/verify-danny → Azure Functions (unavailable on your account)
/api/verify-helen → Azure Functions (unavailable on your account)
```

**After (Cloudflare Workers only):**
```
tamtham.com (Azure SWA)
  ↓
https://tamtham-verify-danny.tamtham.workers.dev → Cloudflare Worker (FREE)
https://tamtham-verify-helen.tamtham.workers.dev → Cloudflare Worker (FREE)
```

**Benefits:**
- ✅ No Azure Functions plan needed (free tier included with Cloudflare)
- ✅ No home server tunnels required (Workers are globally distributed)
- ✅ Same API contract (POST /api/verify-danny/helen)
- ✅ Already using Cloudflare for DNS (single vendor)
- ✅ Simplified infrastructure: Azure for static hosting, Cloudflare for everything else

  <files>src/api/verify-danny/index.js</files>
  <action>
    Create `src/api/verify-danny/index.js` as a simple Cloudflare Worker:

    **Structure:**
    ```javascript
    addEventListener('fetch', event => {
      event.respondWith(handleRequest(event.request));
    });

    async function handleRequest(request) {
      // Handle CORS preflight
      if (request.method === 'OPTIONS') {
        return new Response(null, {
          headers: {
            'Access-Control-Allow-Origin': '*',
            'Access-Control-Allow-Methods': 'POST, OPTIONS',
            'Access-Control-Allow-Headers': 'Content-Type',
          },
        });
      }

      // Only allow POST
      if (request.method !== 'POST') {
        return new Response('Method not allowed', { status: 405 });
      }

      try {
        // Parse request body
        const body = await request.json();
        const { token } = body;

        if (!token) {
          return new Response(
            JSON.stringify({ error: 'Missing token' }),
            { status: 400, headers: { 'Content-Type': 'application/json' } }
          );
        }

        // Validate Turnstile token
        const validation = await fetch('https://challenges.cloudflare.com/turnstile/v0/siteverify', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            secret: env.TURNSTILE_SECRET_KEY,
            response: token,
          }),
        }).then(r => r.json());

        if (validation.success) {
          // Success: redirect to danny subdomain
          return Response.redirect('https://danny.tamtham.com', 302);
        } else {
          // Failure: return 403
          return new Response(
            JSON.stringify({ error: 'Verification failed' }),
            { status: 403, headers: { 'Content-Type': 'application/json' } }
          );
        }
      } catch (error) {
        return new Response(
          JSON.stringify({ error: 'Internal server error' }),
          { status: 500, headers: { 'Content-Type': 'application/json' } }
        );
      }
    }
    ```

    **Environment variable needed:**
    - `TURNSTILE_SECRET_KEY` (from Cloudflare Turnstile dashboard)
