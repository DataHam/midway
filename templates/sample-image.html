Image Usage Patterns

This template demonstrates proper image optimization patterns for web performance and accessibility.

Below-Fold Images

Images below the initial viewport use lazy loading to improve performance:

Best Practices Summary

Hero Images (Above the Fold)

  • fetchpriority="high" - Prioritize loading for better LCP
  • NO loading="lazy" - Must load immediately
  • width/height - Prevents Cumulative Layout Shift (CLS)
  • srcset - 4 variants: 375w, 768w, 1024w, 1920w

Below-Fold Images

  • loading="lazy" - Defers loading until visible
  • decoding="async" - Async image decoding
  • width/height - Prevents CLS
  • srcset + sizes - Responsive with proper breakpoints

General Guidelines

  • All images converted to WebP format for optimal compression
  • Quality set to 80 - Good balance between size and quality
  • Explicit dimensions - Always specify width and height
  • Descriptive alt text - Required for accessibility
  • No Google Fonts CDN - All assets self-hosted
